# Date-task-bot

Простой бот для создания напоминаний о задачах при помощи естественного языка.


## Возможности

- Создание напоминаний с датой выполнения, задаваемой естественным языком, например: "Созвон завтра в 12:00" 
- Напоминания о задаче с интервалами до основного времени выполнения
- Просмотр списка задач и управление им

## Использование

Перед использованием бота необходимо задать часовой пояс командой `/timezone`.

Для создания напоминания, необходимо отправить сообщение боту в соответствии с требованиями, приведенными в следующем разделе. 

После создания напоминания можно проверить корректность распознавания даты и времени, отредактировать и удалить напоминание.

Команда `/tasks` выводит список всех напоминаний с возможностью открыть каждое и отредактировать или удалить.


## Варианты написания

Все задачи должны содержать две части - **текст** и **дату и время** выполнения. Порядок не важен, однако они должны быть в сообщении. Далее текст задачи формируется из текста из сообщения, без даты.

При указании только времени дата будет сегодняшняя, то есть `12:00` означает `сегодня в 12:00`, если текущее время меньше либо равно `12:00`, иначе будет использована дата следующего дня. Такая же логика используется и в других случаях:
- `30 февраля` означает 30 февраля ближайшего високосного года
- `Вторник` - ближайший вторник
- `3 января` - ближайшее 3 января, в следующем году, если на момент создания записи дата позднее 3 января. 

Время на данный момент имеет один вариант записи: `12:34` - часы и минуты разделенные символом двоеточия `:`. Допускаются варианты `09:12` и `9:12` - с ведущим нулем и без. 

Варианты записи относительной даты:
- `завтра` - следующий день
- `послезавтра` - через 1 день (сегодня 1 января, целевой день - 3 января)
- `послепослезавтра` - через 2 дня (сегодня 1 января, целевой день - 4 января)
- `через <x> дней(я)` - текущий день + `x` (через 1 день - завтра) (запись количества может быть как в числах, так и словах (один, два), но рекомендуется использовать числа )
- `через день` - завтра
- `через неделю` - текущий день + 7 дней
- `через <x> недель(ю,и)` - текущий день + (7 * на число недель) дней
- `через месяц` - текущий месяц + 1, число то же
- `через <x> месяц(ев,а)` - текущий месяц + число месяцев, день тот же. 

Если число месяца не существует в целевом месяце, будет выбран последний день месяца. При создании задачи **30 января** `через месяц` дата задачи будет `28/29 февраля`

Запись дня недели:
- `понедельник` - ближайший понедельник
- `пн/вт/ср/чт/пт/сб/вс` - сокращения

При использовании дня недели невозможно использовать относительные обозначения недель - `понедельник через 2 недели` - невалидная строка.

При указании текущего дня недели и любого времени (даже будущего), будет использован день через неделю от текущего. Если текущая дата **08.02, воскресенье, 12:00**, то при записи `воскресенье 18:00` задача будет создана с датой **15.02 18:00**

Запись месяца:
- `январь` - ближайший январь
- `янв/фев/мар/апр/май/июн/июл/авг/сен/окт/ноя/дек` - сокращения

При указании месяца без дня будет использован текущий день. Например, если делать запись `март <текст задачи>` 12 января, то задача будет создана на `12 марта в 00:00`.

Запись года:
- `2026`
- `2026 год`
- `2026 г`
- `2026г`

Точка в `г.` не поддерживается и будет учитываться как часть текста задачи.

Запись дня месяца:
- `2 января` - ближайшее 2 января (текущего или следующего года)
- `2 янв`

Относительное время в часах 
- `через час` - текущий час + 1 час
- `через <x> час(а,ов)` - текущий час + `x` часов
- `через <x> минут` - текущее число минут + `x` минут


Предполагается комбинирование этих обозначений для указания точного времени и даты:
- `1 января 0:30`
- `2 января 12:00 2027 года`
- `через десять дней 12:00`

Примеры корректных задач:
- `Созвон завтра в 12:00`
- `Сдать отчет через 3 дня 18:00`
- `Встреча 3 января 12:00`

## Установка и запуск


### Переменные окружения

Создайте файл `.env` с обязательными переменными:

```bash
BOT_TOKEN=your_token  # Токен бота
```

Необязательные переменные:

```bash
LOG_LEVEL=INFO  # Уровень логирования
```


### Запуск через Docker

Рекомендуется вариант запуска проекта при помощи Docker.

#### Сборка:

```bash
docker build -t date-task-bot .
```

Можно использовать предварительно собранный образ:

```
docker pull ghcr.io/imbalet/date_task_bot:dev
```

#### Создание volume для базы данных:

```bash
docker volume create database
```

#### Запуск:

```bash
docker run \
    -d \
    --env-file .env \
    -v database:/app/database \
    --name date-task-bot date-task-bot:latest
```
или для готового образа:
```bash
docker run \
    -d \
    --env-file .env \
    -v database:/app/database \
    --name date-task-bot ghcr.io/imbalet/date_task_bot:dev
```


### Локальный запуск


#### uv

Подробнее про uv: [docs.astral.sh/uv](https://docs.astral.sh/uv/)

1. Установите зависимости и пакет:

```bash
uv sync
```

1. Запустите проект:

```bash
uv run src/date_task_bot/main.py
```


#### pip

1. Создайте виртуальное окружение и активируйте его:

```bash
python -m venv .venv
source ./.venv/bin/activate
```

2. Установите зависимости:

```bash
pip install .
```

1. Запустите проект:

```bash
python src/date_task_bot/main.py
```


## Тестирование

Запуск тестов с `uv`:

```bash
uv sync --all-groups
uv run -m pytest tests
```

Для `pip` необходимо вручную установить тестовые зависимости:

```bash
pip install pytest>=8.4.2 pytest-asyncio>=1.2.0 pytest-cov>=7.0.0 pytest-mock>=3.15.1
```

Запуск тестов:

```bash
pytest tests
```
